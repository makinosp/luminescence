# Personas

## P1: Mobile User

| Attribute               | Description                                                                                                  |
| ----------------------- | ------------------------------------------------------------------------------------------------------------ |
| **Title**               | Individual User Managing Home Finances on the Go                                                             |
| **Device**              | iOS / Android Smartphone                                                                                     |
| **Proficiency**         | Average smartphone user with interest in home finance management but average financial knowledge.            |
| **Goals**               | Want to view and record transactions anytime, anywhere. Wants to immediately record expenses while shopping. |
| **Pain Points**         | Annoyed by data input on a small screen. Anxious about inability to operate offline.                         |
| **Key Needs**           | Intuitive UI, fast transaction input, push notification reminders, fingerprint/Face ID authentication.       |
| **Interaction Pattern** | Touch-based interaction. Short duration, high-frequency usage (1-3 minutes per session).                     |

---

## P2: Web User

| Attribute               | Description                                                                                                                               |
| ----------------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| **Title**               | Individual User Managing Home Finances from Desktop Browser                                                                               |
| **Device**              | Desktop/Laptop (Latest Browser)                                                                                                           |
| **Proficiency**         | Average web user interested in home finance analysis with moderate financial literacy.                                                    |
| **Goals**               | Want to understand overall household budget. Want to analyze spending trends through reports and graphs.                                  |
| **Pain Points**         | Tedious manual organization of large transaction data. Complicated categorization process.                                                |
| **Key Needs**           | Responsive Web UI, detailed reports and visualizations, bulk operations (editing multiple transactions), search and filter functionality. |
| **Interaction Pattern** | Mouse and keyboard operations. Medium frequency, extended usage (multiple times per week, 15-30 minutes per session).                     |

---

## P3: CLI User

| Attribute               | Description                                                                                                                                       |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Title**               | Power User Operating Efficiently from the Terminal                                                                                                |
| **Device**              | Terminal Environment (macOS/Linux/Windows)                                                                                                        |
| **Proficiency**         | Developer or system administrator-level technical skills. Proficient in shell operations.                                                         |
| **Goals**               | Want to automate and batch process through scripts. Home finance management integrated into shell workflows.                                      |
| **Pain Points**         | GUI tools are slow to operate. Unable to automate or perform pipeline processing.                                                                 |
| **Key Needs**           | Interactive mode (REPL-like) and script mode both supported, JSON/CSV output support, integration with standard Unix tools (grep, jq, awk, etc.). |
| **Interaction Pattern** | Keyboard operations only. Low frequency but high-volume processing per operation (cron-scheduled execution, batch imports, etc.).                 |

---

## P4: New User

| Attribute               | Description                                                                                                     |
| ----------------------- | --------------------------------------------------------------------------------------------------------------- |
| **Title**               | User Performing Initial Setup (applicable to all of P1/P2/P3)                                                   |
| **Device**              | Any Client (Initial Setup Only)                                                                                 |
| **Proficiency**         | All Levels (Beginner to Advanced)                                                                               |
| **Goals**               | Want to connect to Firefly III server and begin using it in the minimum number of steps.                        |
| **Pain Points**         | Complex setup procedures cause abandonment. Unclear meaning of API tokens.                                      |
| **Key Needs**           | Clear initial setup wizard, immediate feedback on server connection verification, secure configuration storage. |
| **Interaction Pattern** | One-time setup operation. Subsequent use transitions to normal operation.                                       |
